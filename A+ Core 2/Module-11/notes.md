# Key Concepts

## Standard Operating Procedures (SOPs)

Organizations rely on written documentation to ensure IT services are delivered consistently.

Three important documents are:

### Policies

Policies define the organization's overall intentions and security expectations.

### Standard Operating Procedures (SOPs)

SOPs provide detailed step-by-step instructions for performing specific tasks consistently.

Examples include:

* Software installation
* New employee onboarding
* User account creation
* Employee offboarding
* Device deployment
* Access management

### Guidelines

Guidelines provide recommendations when a formal procedure doesn't exist or when situations require professional judgement.

---

## Service Level Agreements (SLAs)

An SLA defines the level of service expected between an organization and either an internal department or an external provider.

Typical SLA components include:

* Expected uptime
* Response times
* Resolution times
* Performance metrics
* Escalation procedures
* Compensation if service levels are not met

I also learned how uptime is measured using the "nines" system (99.9%, 99.99%, etc.), where higher availability means significantly less downtime each year.

---

## Incident and Ticket Management

A ticketing system tracks support requests from creation until resolution.

The general workflow includes:

1. User submits a request.
2. Technician records user and device information.
3. Issue is categorized and prioritized.
4. Initial troubleshooting begins.
5. Ticket is resolved or escalated if necessary.
6. Resolution is documented.
7. Ticket is closed.

Accurate documentation throughout the process is essential for future troubleshooting and reporting.

---

## Ticket Categories and Severity

Support requests generally fall into three categories:

### Requests

Routine, repeatable services such as:

* Password resets
* New user accounts
* Software installation
* Hardware requests

### Incidents

Unexpected interruptions affecting users or services.

### Problems

The underlying root cause responsible for one or more incidents.

---

### Severity Levels

Tickets are prioritized according to business impact.

Common levels include:

* Critical
* Major
* Minor

Higher severity incidents require faster responses and often trigger management notifications.

---

## Escalation Procedures

If an issue cannot be resolved by the first technician, it follows an escalation path.

Typical support tiers include:

### Tier 0

Self-service resources such as FAQs and knowledge bases.

### Tier 1

Help Desk technicians performing initial diagnosis.

### Tier 2

Experienced technicians handling more complex technical issues.

### Tier 3

Engineers, developers, or senior specialists responsible for advanced troubleshooting.

---

## Documentation and Knowledge Bases

Good documentation improves consistency and reduces resolution times.

Useful documentation includes:

* Product documentation
* Configuration guides
* Troubleshooting procedures
* Frequently Asked Questions (FAQs)
* Previous incident resolutions
* Lessons learned reports

Knowledge bases also allow users to solve common problems independently.

---

## Lessons Learned

Major incidents often generate an After Action Report (AAR) or Lessons Learned document.

These reports help organizations:

* Identify root causes
* Improve procedures
* Update SOPs
* Prevent future incidents
* Expand the knowledge base

# Professional Communication

Technical knowledge alone is not enough—effective communication is equally important for IT professionals.

---

## Setting Expectations

When assisting users, technicians should:

* Acknowledge the issue
* Explain the next steps
* Provide realistic timelines
* Communicate delays honestly
* Keep users informed throughout the process

Managing expectations improves customer satisfaction even when problems take longer to resolve.

---

## Professional Documentation

Support tickets should contain clear, accurate information.

Typical sections include:

* Issue description
* Progress notes
* Final resolution

Documentation should be:

* Clear
* Concise
* Professional
* Free from unnecessary technical jargon

---

## Active Listening

Active listening involves giving the customer full attention before proposing solutions.

Good techniques include:

* Asking clarifying questions
* Summarizing the issue
* Taking notes
* Confirming understanding

---

## Questioning Techniques

Two common question types are:

### Open-ended

Encourage users to explain the problem in detail.

Example:

> "Can you describe what happened?"

### Closed

Used to confirm specific information.

Example:

> "Did the error appear after restarting?"

Using both question types helps gather accurate troubleshooting information.

---

## Handling Difficult Situations

Support professionals should:

* Stay calm
* Remain professional
* Avoid arguments
* Show empathy
* Focus on solving the problem
* Avoid taking complaints personally

When customers become abusive, organizations may follow escalation policies or involve supervisors.

---

## Professional Conduct

Important professional behaviors include:

* Respecting customer privacy
* Protecting confidential information
* Dressing appropriately
* Using professional language
* Demonstrating cultural awareness
* Following organizational policies

# Operating Systems

The module introduced the major operating systems commonly encountered in IT support.

---

## Microsoft Windows

Windows supports:

* Home users
* Business workstations
* Servers
* Tablets

Business environments commonly use:

* Windows 10
* Windows 11
* Windows Server

---

## Apple macOS

macOS is Apple's desktop operating system.

Characteristics include:

* UNIX-based
* Exclusive to Apple hardware
* Strong integration with Apple devices
* Regular free updates

---

## Linux

Linux is an open-source operating system available in many distributions.

Popular distributions include:

* Ubuntu
* Debian
* Fedora
* Red Hat Enterprise Linux (RHEL)
* SUSE
* Linux Mint
* Arch Linux

Linux dominates many server environments and is widely used in cloud computing, networking, and cybersecurity.

---

## UNIX

UNIX introduced many concepts later adopted by Linux and macOS.

It is known for:

* Stability
* Portability
* Multi-user capabilities
* Command-line administration

---

## ChromeOS

ChromeOS is Google's lightweight operating system designed primarily for web-based workloads.

Common devices include:

* Chromebooks
* Chromeboxes

---

## Mobile Operating Systems

### Apple iOS / iPadOS

Designed exclusively for Apple mobile devices.

### Android

Based on Linux and developed by Google through the Open Handset Alliance.

Android powers smartphones and tablets from many manufacturers.

# File Systems

## Windows File Systems

### NTFS

Primary Windows file system featuring:

* Journaling
* Security permissions
* Encryption (EFS)
* Shadow Copies
* Large volume support

---

### ReFS

Resilient File System focuses on:

* Data integrity
* Scalability
* Fault tolerance
* High availability

---

### FAT32

Older file system offering excellent compatibility but limited by:

* 4 GB maximum file size
* Limited security features

---

### exFAT

Designed for removable storage devices.

Benefits include:

* Large file support
* Cross-platform compatibility
* Commonly used on flash drives and SD cards

---

## Linux and macOS File Systems

### Linux

Common Linux file systems include:

* ext4
* XFS

### macOS

Modern Apple devices use:

* APFS (Apple File System)

APFS supports:

* Snapshots
* Encryption
* Fast file operations
* SSD optimization

---

# Operating System Compatibility

Before deploying or upgrading an operating system, technicians should evaluate:

* Hardware compatibility
* Software compatibility
* Driver support
* Network compatibility
* User training requirements

Modern operating systems may require hardware features such as TPM 2.0 and Secure Boot.

---

## Vendor Lifecycle

Every operating system eventually reaches End of Life (EOL).

Lifecycle stages generally include:

* Beta testing
* Active support
* Extended support
* End of Life (EOL)

Running unsupported operating systems increases security risks because they no longer receive updates or security patches.

# Key Takeaways

* SOPs and policies help organizations deliver consistent and secure IT services.
* Ticketing systems document the entire support lifecycle from incident creation to resolution.
* Proper documentation and knowledge bases improve troubleshooting efficiency and reduce repeated work.
* Professional communication is just as important as technical knowledge for IT support professionals.
* Active listening and effective questioning help technicians diagnose issues more accurately.
* Windows, macOS, Linux, ChromeOS, Android, and iOS each serve different markets and use cases.
* Understanding file systems such as NTFS, ReFS, FAT32, exFAT, ext4, XFS, and APFS is essential for supporting different operating systems.
* Compatibility planning and vendor lifecycle management help organizations maintain secure and reliable IT environments.
